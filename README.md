Password Generation
Идея: 
  Создать простой, красивый и понятный генератор безопасных паролей, для защиты от взлома и утечки данных из аккаунтов

Основной функционал:
  ~ Генерация пароля - Решает главную задачу — создать случайный пароль - Основной функционал
  ~ Выбор длины пароля - Позволяет настроить пароль под требования сайта - Основной функционал
  ~ Копирование пароля - Даёт сразу использовать результат - Важный функционал
  ~ Смена фона - Создаёт атмосферу, но не влияет на работу - Дополнительный функционал
  ~ Цитата о грибах - Эмоциональный элемент, бренд - Дополнительный функционал

      JavaScript (ES6+)
      CSS3
    UI/UX & Ассеты
      Google Fonts (Cinzel, Cormorant Garamond)

      Браузерные Web API

Архитектура:
  [Браузер] 
   │ (HTTP GET /)
   ▼
  [Flask] → отдаёт templates/index.html + static/style.css + static/images/*
   │
   │ (POST /generate, JSON: {"length": 12})
   ▼
  [Python генератор] → random.choice() из ascii_letters+digits+punctuation
   │
   │ (JSON: {"password": "xK9#mP2!"})
   ▼
  [JS на клиенте] → вставляет пароль, меняет фон, активирует кнопку "Копировать"

  Инструкция запуска:
️    Требования
      Перед началом убедиться, что установлены:
        Python 3.10+ (Проверь командой: python3 --version)
        Git (Если скачиваешь через git) или просто доступ к папке с файлами.
    Шаг 1: Подготовка структуры
      Убедиться, что файлы лежат в следующих папках (это критично для работы):
        hom_project_Pass_Gen_Code/
        ├── app.py
        ├── README.md
        ├── static/
        │   ├── style.css
        │   └── images/
        │       ├── bg1.jpg
        │       ├── bg2.jpg
        │       ├── bg3.jpg
        │       ├── bg4.jpg
        │       └── bg5.jpg
        └── templates/
            └── index.html
    Шаг 2: Создание виртуального окружения (рекомендуется)
      Чтобы не засорять систему библиотеками, создайте изолированную среду:
        # Для Linux/macOS:
          python3 -m venv venv
          source venv/bin/activate
        # Для Windows:
          python -m venv venv
          venv\Scripts\activate
    Шаг 3: Установка зависимостей
      Установить Flask — единственный внешний пакет проекта:
        pip install flask
    Шаг 4: Запуск сервера
      Запустить приложение:
        python app.py
      Должен увидеть сообщение: * Running on http://127.0.0.1:5000
    Шаг 5: Открытие в браузере
      Открой любую ссылку из лога (обычно http://127.0.0.1:5000) в браузере.
